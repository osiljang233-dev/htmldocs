AI 자동화 운영 전략 보고서 1장

AI-OPERATION-STRATEGY · 2026-07-13 · AI 에이전트 역할 분리 · 협업 구조 설계 · 칸반보드 활용 방안
TL;DR
  • AI가 완전 자동으로 할 것 / AI 도움받아 사람이 처리할 것 / 사람이 직접 해야 할 것 — 3단계로 역할 분리
  • 협업 시스템이 필요한 작업 vs 단독 에이전트로 충분한 작업 명확히 구분
  • 칸반보드는 시스템이 잘 돌아갈 때는 텔레그램 결과물만 확인하면 충분 — 필요할 때만 여는 도구
SECTION 01 AI 에이전트 역할 분리 설계 ▶
① AI가 완전 자동으로 하는 것
  • 매일 지정 시간 경기 데이터 수집 — ESPN, MLB Stats, Baseball Savant, 파워랭킹, 헤드라인 뉴스
  • 경기 분석 + 픽 결정 + Confidence 5단계 산출
  • 나레이션 생성, 자막 싱크, BGM 믹싱 → 영상 편집 → final.mp4 완성
  • 텔레그램으로 완성본 자동 전송 / 매일 지정 시간 크론 자동 실행
② AI 도움을 받아 사람이 빠르게 처리하는 것
  • 스크립트 검토/확정 — AI 초안 작성, 사람이 5분 검토. 직접 쓰면 1시간 → 5분으로 단축
  • 블로그 포스팅 검수 — AI 초안, 사람이 사실 확인 + 톤 조정 + SEO 키워드 반영 여부 체크
  • 유튜브 제목/설명/태그 — AI 자동 생성, 사람이 30초 확인 후 업로드
③ 사람이 직접 해야 하는 것
✅ 필수 — AI 대체 불가
작업이유구분
영상 최종 확인 저작권 이슈, 싱크 오류, 부적절한 장면 — AI가 판단 불가한 영역 필수
픽 최종 동의 AI 픽이 틀렸을 때 채널 신뢰도 직결. 최종 책임은 사람이 짐 필수
유튜브 업로드 채널 계정 접근 권한, 최종 발행 책임 필수
⚡ 선택 — 속도 vs 품질 저울질
작업AI로 할 때사람이 직접 할 때비고
영상 소스 선정 빠르지만 자막 싱크 안 맞는 영상 나올 수 있음 느리지만 영상 품질 확실히 올라감 속도↑ 품질↑
썸네일 제작 무난한 수준, 자동화 가능 클릭률 차이 날 수 있음 선택
SECTION 02 협업 시스템 vs 단독 에이전트 ▶
🤝 협업 시스템이 필요한 작업

MLB 분석 + 영상 제작

리서치 → 분석 → 콘텐츠 → 영상 단계별 QA 필수. 한 단계 오류가 전체에 영향. 픽이 틀렸을 때 채널 신뢰도 직결

SEO 블로그 운영

키워드 선정 → 작성 → 검수 → 발행 → 성과 모니터링. 단독으로 시키면 퀄리티 들쭉날쭉

🤖 단독 에이전트로 충분한 작업

경쟁사 콘텐츠 모니터링

유튜브 채널, 네이버/구글 상위 노출 변화 매일 체크 + 요약 리포트. 유료 구독 콘텐츠도 계정 정보 제공 시 자동 수집 가능

키워드 순위 추적

목표 키워드("MLB 픽", "야구 베팅 분석" 등)가 구글/네이버에서 몇 위인지 매일 자동 체크. 순위 5위 이상 급락 시 이상 징후 알림

예) "다음엔 다저스 해줘요" → 콘텐츠 방향 참고
"픽 틀렸잖아요 ㅋㅋ" → 픽 정확도 피드백
"나레이션이 너무 빨라요" → 영상 품질 개선 힌트

신규 키워드 발굴

구글 트렌드·네이버 실시간·유튜브 급상승 영상을 매주 자동 스캔. 우리 주제와 연관된 키워드만 필터링 후 검색량·경쟁 강도 붙여서 우선순위 리스트로 텔레그램 전송.

예) MLB·야구 베팅·스포츠 분석 등 설정한 주제에 맞게 필터 조건 변경 가능
핵심 판단 기준

결과물이 틀렸을 때 큰 영향이 있는가? → 협업+QA 필수 / 참고용이라 QA가 불필요한가? → 단독 에이전트

SECTION 03 칸반보드 활용 방안 ▶

헤르메스 내 태스크 관리 시스템. 각 작업이 어느 단계에 있는지 실시간으로 확인할 수 있는 대시보드.
시스템이 잘 돌아갈 때는 텔레그램 결과물만 확인하면 충분 — 칸반보드는 필요할 때만 여는 도구.

상황 칸반보드 있을 때 없을 때 대안
오류 발생 대시보드에서 어느 단계인지 즉시 파악 텔레그램에서 매니저한테 "어디서 멈췄어?" 물어보기
진행 상황 실시간 단계별 확인 텔레그램에서 매니저한테 "지금 몇 단계야?" 물어보기
병목 구간 단계별 소요시간 데이터로 정확히 파악 매니저한테 물어봐도 되지만 누적 데이터 없어 부정확
동시 작업 3개 경기 진행상황 한눈에 정리 매니저한테 A, B, C 각각 따로 물어보기
직원 성능 진행 중 실시간 모니터링·즉시 개선 가능 결과물 나온 후 매니저한테 사후 확인만 가능

작업 규모가 커질수록 — 하루 3개 이상 영상, 블로그+쇼츠 동시 운영 — 칸반보드 없이는 관리가 점점 어려워진다.
