  • 6개 링크사이트 블로그 카테고리에 매일 경기 분석글을 자동 발행한다.
  • 베팅 관심도 기준 TOP6 경기를 선정해 각 사이트에 1개씩 배분한다.
  • 날짜+팀명 롱테일 키워드 틈새를 공략해 구글 SEO 유입을 확보한다.

01 프로젝트 개요

6개 링크사이트에 블로그 카테고리를 만들고 매일 스포츠 경기 분석글을 자동으로 생성한다. 베팅에 관심있는 유저가 구글에서 경기 분석을 검색할 때 유입되는 구조를 만드는 것이 목적이다.

아무것도 없는 사이트보다 매일 글이 올라오는 사이트가 구글 크롤러가 자주 방문하고 도메인 신뢰도가 쌓인다. 초반엔 유입이 적어도 3~6개월 꾸준히 쌓이면 달라진다.

이 프로젝트는 랜딩페이지 프로젝트와 별개로 운영되지만, 나중에 랜딩페이지와 연계해 시너지를 낼 수 있다.

03 TOP6 선정 로직

실제 베팅량은 북메이커가 공개하지 않는다. 대신 베팅 관심도를 나타내는 대리 지표를 조합해 점수화한다.

Rule Engine이 점수를 계산하고 상위 10개를 추린 뒤, AI가 최종 6개를 확정한다. Rule만 쓰면 돌발 이슈를 못 잡고, AI만 쓰면 매일 결과가 흔들리기 때문에 하이브리드 방식을 쓴다.

04 데이터 수집

글 품질은 데이터 품질에서 결정된다. 실제 수치가 들어간 글과 없는 글의 차이는 크다. MLB/NBA는 무료 공식 API로 안정적으로 수집하고, 나머지는 크롤링으로 보완한다.

BeautifulSoup이란? 파이썬 크롤링 라이브러리다. 웹사이트 HTML을 자동으로 긁어와서 원하는 데이터만 뽑아내는 도구. 예를 들어 네이버스포츠에서 KBO 오늘 경기 결과를 자동으로 가져올 때 사용한다. 크롤링 = BeautifulSoup으로 이해하면 된다.

05 글 생성 파이프라인

프롬프트는 고정 템플릿, 데이터만 매일 자동으로 채워 넣는 구조다. Claude API를 2~3회 호출로 제한해 단순하게 유지한다.

Step 1 데이터 수집 오늘 전체 경기 + 오즈 + 스탯 + 부상자 + 뉴스 수집
Step 2 TOP6 선정 Rule 점수화 → 상위 10개 → AI 최종 6개 확정
Step 3 프롬프트 조립 고정 템플릿에 오늘 실제 데이터 자동 삽입
Step 4 Claude API 호출 경기별 분석글 생성 (6개 경기 × 1회 호출)
Step 5 파일 저장 Orca에서 글 파일로 저장 → 검토 후 수동 업로드 (초반)
프롬프트 구조 (개념)

프롬프트는 세 레이어로 구성된다. 실제 프롬프트는 데이터 수집 테스트 후 실제 데이터 형태에 맞게 정교하게 다듬는다.

레이어 1. 역할 설정 (고정) 너는 스포츠 경기 분석 전문가야. 베팅에 관심있는 한국 성인 독자가 읽는다. 데이터 기반으로 분석하되 읽기 쉽게 써줘. 전문 용어는 간단히 설명을 달아줘. 레이어 2. 실제 데이터 (매일 자동 삽입) → 양팀 최근 5경기 스탯 → 선발 투수 ERA + 최근 등판 성적 → 부상자 현황 → 오즈 + 개장 대비 무브먼트 → 오늘 주요 뉴스 헤드라인 레이어 3. 출력 형식 (고정) → SEO 최적화 H1 제목 (날짜 포함) → 경기 기본 정보 → 핵심 요약 (3줄 이내) → 양팀 폼 분석 → 선발/주요 선수 분석 → 오즈 분석 → 종합 예측 → FAQ 5개 (롱테일 키워드 포함) → 텔레그램 참여 CTA
실제 프롬프트는 나중에: 지금 단계에서 완성된 프롬프트를 확정하는 건 의미없다. 실제 데이터가 어떤 형태로 들어오는지 테스트해보고 나서 정교하게 다듬는다. 프롬프트 품질이 글 품질을 결정하기 때문에 가장 공을 들여야 하는 부분이다.
06 SEO 전략

네이버, 다음, ESPN 같은 대형 포털은 빅키워드를 다 잡고 있다. 신규 사이트가 쓸 수 있는 SEO 전략은 여러 가지이며, 그 중 하나가 날짜+팀명 롱테일 키워드 틈새 공략이다. Ahrefs 조사 후 실제 검색량 데이터를 보고 어떤 전략을 메인으로 가져갈지 결정한다.

전략 1. 날짜+팀명 H1 제목 공식 (가설 — Ahrefs 검증 필요)
[팀A] vs [팀B] 오늘 경기 분석 예측 [YYYY.MM.DD] 예시) 다저스 vs 양키스 오늘 경기 분석 예측 2026.07.22 맨시티 vs 레알마드리드 챔스 경기 분석 2026.07.22 두산 vs LG 오늘 경기 분석 예측 2026.07.22

날짜를 제목에 반드시 넣어야 한다. 매일 새 글이 날짜별로 구글에 색인되는 구조가 된다. 같은 팀 매치업이라도 날짜가 다르면 다른 키워드로 잡힌다.

전략 2. FAQ 롱테일 키워드 (글 하나에서 여러 키워드 동시 공략)

FAQ 5개 안에 롱테일 키워드를 전략적으로 배치하면 글 하나에서 여러 키워드가 동시에 잡힌다.

FAQ 예시잡히는 키워드
"다저스 오늘 선발 누구야?"다저스 오늘 선발, 다저스 선발 투수
"양키스 오늘 경기 몇 시야?"양키스 오늘 경기 시간, 양키스 중계
"다저스 양키스 오즈 어때?"다저스 양키스 오즈, MLB 오즈 분석
"오타니 오늘 나와?"오타니 오늘 출전, 오타니 선발
"다저스 요즘 폼 어때?"다저스 최근 성적, 다저스 폼

FAQ 키워드 풀은 Ahrefs로 사전에 리그별로 만들어놓고 프롬프트에 고정 탑재한다.

전략 3. 꾸준한 발행으로 도메인 권위 축적

아무것도 없는 사이트 → 구글이 크롤링 안 함 → 색인 안 됨 → 존재 자체가 없음

매일 글이 올라오는 사이트 → 구글 크롤러가 매일 방문 → 신선한 콘텐츠 신호 → 도메인 점수 상승 → 새 글이 더 빨리 색인됨

초반엔 유입이 거의 없어도 3~6개월 꾸준히 쌓이면 달라진다. 완벽한 글 1개보다 꾸준한 글 100개가 SEO에서 이긴다.

07 Ahrefs 활용

Ahrefs는 글 쓸 때마다 돌리는 게 아니다. 시스템 설계할 때 한 번만 제대로 조사하고, 그 결과를 프롬프트 템플릿에 고정 탑재한다. 이후 자동화는 그 설계 위에서 돌아간다.

중요: 아래 키워드 예시는 가설 기반이다. 실제 검색량과 KD는 반드시 Ahrefs를 직접 돌려서 검증한 뒤 확정해야 한다.
Step 1 날짜형 키워드 검증 "팀A 팀B 분석" 패턴의 날짜 포함 변형 키워드 검색량 + KD 확인
Step 2 경쟁사 역추적 구글 1페이지 소형 블로그 도메인 → Site Explorer → Organic keywords 확인
Step 3 H1 공식 확정 KD 낮고 검색량 있는 제목 패턴 찾아서 공식화
Step 4 FAQ 키워드 풀 구축 리그별 롱테일 키워드 20개씩 → 프롬프트 고정 탑재
KD 기준: 신규 사이트는 KD 10 이하 키워드부터 공략. KD 20 이상은 사실상 불가능.
08 E-E-A-T 전략

구글은 AI가 썼는지보다 유용한가를 기준으로 평가한다. 실제 데이터가 들어간 글은 다른 블로그와 차별화되고 E-E-A-T 기준을 어느 정도 충족한다.

기준우리 글의 충족 방식수준
Experience 실제 오즈 데이터 + 최근 스탯 기반 분석 중간
Expertise 오즈 무브먼트 + 선발 ERA + 팀 폼 분석 포함 중간
Authoritativeness 공식 API + OddsPortal 출처 명시 충족
Trustworthiness 날짜, 실제 수치, 선발 라인업 기반 충족
우리 목표는 네이버 스포츠를 이기는 게 아니다. 대형 포털이 커버하지 못하는 틈새 키워드를 매일 꾸준히 채워나가는 것이 전략이다. 정확한 틈새 키워드는 Ahrefs 조사 후 확정한다.
